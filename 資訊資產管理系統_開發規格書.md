## 1. 專案概述

### 1.1 專案背景

為有效管理公司資訊資產，提升盤點效率與資產追蹤準確性，規劃建置資訊資產管理系統。本系統將整合 ERP 資產資料，並透過影像辨識技術實現資產標籤自動識別，大幅降低人工盤點作業負擔。

### 1.2 專案目標

- 建立完整的資訊資產生命週期管理機制
- 與 ERP 系統整合，確保資產資料一致性
- 導入影像辨識技術，提升盤點作業效率
- 提供即時資產狀態查詢與統計報表
- 建立完善的權限控管機制

### 1.3 系統範圍

| 項目 | 說明 |
|------|------|
| 系統類型 | Web 應用程式（RWD 響應式設計） |
| 部署方式 | 雲端 SaaS 部署 |
| 整合系統 | ERP 系統（透過 API 整合） |
| 使用者範圍 | 資訊部門、財務部門、各部門資產管理人員 |
| 架構模式 | 前後端整合，以角色權限區分功能 |

**盤點流程：**

```
1. 開啟相機拍攝標籤
         │
         ▼
2. OCR 自動辨識標籤內容
         │
         ▼
3. 擷取資產編號、類別、名稱、取得年月
         │
         ▼
4. 自動比對系統資產清單
         │
         ▼
5. 顯示比對結果
         │
         ▼
6. 確認或修正後存檔
```

**標籤辨識規格（基於現有標籤格式）：**

| 欄位 | 辨識規則 (Regex) | 範例 |
|------|------------------|------|
| 資編 | `資編[：:]\s*(\d{9})` | 資編：040400275 |
| 類別 | `類別[：:]\s*(.+-.+)` | 類別：資訊-可攜式電腦 |
| 名稱 | `名稱[：:]\s*(.+)` | 名稱：ASUS筆記型電腦 |
| 取得年月 | `取得年月[：:]\s*(\d{4}/\d{1,2})` | 取得年月：2023/9 |

**自動匹配邏輯：**

```
辨識結果
    │
    ▼
擷取「資編」欄位
    │
    ▼
查詢系統資產 DB (WHERE asset_no = ?)
    │
    ├─── 找到 ──→ 比對其他欄位 ──┬─ 一致 ──→ 標記已盤點
    │                            │
    │                            └─ 不一致 → 列入差異報表
    │
    └─── 找不到 → 標記為盤盈/異常
```

## 5. API 規格

### 5.1 API 設計原則

- RESTful 設計風格
- 統一回應格式
- JWT Token 認證
- 版本控制 (`/api/v1/`)

### 5.2 統一回應格式

**成功回應：**
```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}
```

**失敗回應：**
```json
{
  "success": false,
  "error": {
    "code": "ERR_001",
    "message": "錯誤訊息"
  }
}
```

**分頁回應：**
```json
{
  "success": true,
  "data": [ ... ],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 100,
    "totalPages": 5
  }
}
```

## 7. 非功能性需求

### 7.1 效能需求

| 項目 | 需求 |
|------|------|
| 一般查詢回應時間 | < 2 秒 |
| 複雜報表產生時間 | < 10 秒 |
| OCR 辨識回應時間 | < 5 秒 |
| 同時在線使用者 | 支援 100 人以上 |
| 資料筆數 | 支援 100 萬筆資產 |

### 7.2 可用性需求

| 項目 | 需求 |
|------|------|
| 系統可用率 | 99.5% 以上 |
| 計畫性維護時間 | 每月 < 4 小時 |
| 非計畫停機時間 | 每月 < 1 小時 |

### 7.3 安全性需求

| 項目 | 需求 |
|------|------|
| 傳輸加密 | HTTPS (TLS 1.3) |
| 密碼儲存 | bcrypt 加密 |
| 認證機制 | JWT Token (有效期 8 小時) |
| 操作日誌 | 完整記錄所有異動操作 |
| 權限控管 | RBAC 角色權限模型 |

### 7.4 相容性需求

| 項目 | 支援範圍 |
|------|----------|
| 桌面瀏覽器 | Chrome、Edge、Safari 最新兩個版本 |
| 行動裝置 | iOS 15+、Android 12+ |
| 螢幕解析度 | 1366x768 以上 |
| 響應式設計 | 支援 320px ~ 1920px 寬度 |

### 7.5 備份與災難復原

| 項目 | 需求 |
|------|------|
| 資料庫備份 | 每日完整備份 + 每小時增量備份 |
| 備份保留 | 30 天 |
| 異地備援 | 不同區域 (Region) 備援 |
| RTO | < 4 小時 |
| RPO | < 1 小時 |
